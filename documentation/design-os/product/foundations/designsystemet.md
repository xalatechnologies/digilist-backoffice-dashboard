Design system: Designsystemet.no (Digdir)

Rules:
- Use Designsystemet.no typography, spacing, and components as default.
- Do not introduce custom component styles unless necessary for domain-specific UI.
- Prefer existing patterns for:
  - Tables
  - Badges/Tags
  - Buttons
  - Alerts
  - Cards
  - Lists
  - Icons

Accessibility:
- WCAG 2.1 AA
- Keyboard navigation required for all interactive elements
- Focus states must be visible and consistent
- No color-only communication (status must have text + icon where useful)

Admin UI principles:
- Calm density: information-rich but not noisy
- Consistent alignment and spacing
- Clear hierarchy: headline → KPIs → primary work list → recent events
